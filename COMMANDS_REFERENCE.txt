# FICHIER DE RÉFÉRENCE - COMMANDES UTILES

Ce fichier contient une liste de commandes utiles que vous pouvez exécuter dans le terminal de votre station de travail pour inspecter et surveiller les ressources de votre application.

---

### 1. ANALYSE DE L'ESPACE DISQUE (Poids du projet)

--
# Affiche la taille totale du répertoire de votre projet (lisible par l'homme, ex: 250M)
du -sh .
--

--
# Affiche la taille de chaque dossier et fichier à la racine du projet, trié du plus grand au plus petit.
# Très utile pour identifier les dossiers lourds comme `node_modules` ou `.next`.
du -sh * | sort -rh
--

--
# Affiche la taille spécifique du dossier des dépendances.
du -sh node_modules
--

--
# Affiche la taille spécifique du cache de build de Next.js.
du -sh .next
--

---

### 2. ANALYSE DE LA MÉMOIRE (RAM)

--
# Affiche la mémoire totale, utilisée, libre, partagée et en cache du système.
# L'option -h la rend lisible (ex: 1.2G, 512M).
free -h
--

--
# Lance un moniteur de ressources interactif en temps réel.
# C'est l'outil le plus puissant pour voir quels processus (comme `node` pour Next.js)
# consomment le plus de CPU et de mémoire.
# (Peut nécessiter une installation: sudo apt-get install htop)
htop
--

---

### 3. GESTION DU CACHE

--
# GESTION DU CACHE NPM (Dépendances)

# Vérifie l'intégrité du cache npm, supprime les données corrompues et affiche un résumé.
npm cache verify
--

--
# GESTION DU CACHE REDIS (Si accessible)

# Se connecte à l'interface de ligne de commande de Redis.
# L'URL de connexion est nécessaire si Redis n'est pas sur localhost.
redis-cli -u <VOTRE_REDIS_URL>

# Une fois connecté à redis-cli, cette commande affiche des informations détaillées sur l'utilisation de la mémoire par Redis.
INFO memory

# Affiche toutes les clés actuellement stockées dans Redis. Utile pour le débogage.
KEYS *
--

---

### 4. PROCESSUS ET RÉSEAU

--
# Liste les processus en cours d'exécution avec des détails.
# Utile pour trouver l'ID du processus (PID) de votre serveur Next.js.
ps aux
--

--
# Affiche les processus qui écoutent sur des ports réseau.
# Très utile pour vérifier sur quel port votre application tourne.
# Remplacez 9002 par le port de votre application si nécessaire.
sudo lsof -i :9002
--
